================================================================================
2. FILE-LEVEL DIRECTIVES
================================================================================

  Directives must appear at the TOP of the file, BEFORE any code.
  Enforced order: #mode -> #autoFree -> #allowMemoryLeaks
  #switchBreakMode can appear before any switch statement.

  Placing directives after code is a compile error.
  Each directive can only be declared once per file.

--------------------------------------------------------------------------------
MODE DIRECTIVES
--------------------------------------------------------------------------------

  #mode static;
      Full static typing.
      - All variables must have explicit type annotations.
      - All functions must declare a return type or void.
      - All function parameters must have type annotations.
        (main() is exempt from param type requirements)
      - Non-void functions must have a return statement.
      - All code paths must return a value.
      - Return type must match declared type (hard error, not warning).
      - Untyped bare assignment is a hard error: x = 5; -> ERROR.
      - void functions cannot be used in expressions.

  #mode dynamic;
      Full dynamic mode. No type checking whatsoever.
      - int x = "hello"; is fine (annotation ignored at runtime).
      - Any variable can hold any value.
      - Arity is still checked at compile time.

  #mode both;    (DEFAULT if no #mode directive)
      Mixed mode.
      - Typed variables are type-checked.
      - Untyped variables are fully dynamic.
      - int x = "hello"; -> ERROR (typed var, wrong type).
      - x = "hello"; -> OK (untyped, dynamic).
      - Functions with declared return types get a WARNING (not error)
        if return type cannot be verified on all paths.

  Implementation note: #mode is pre-stripped from source before lexing.
  The lexer never sees it as a token. #autoFree and #allowMemoryLeaks are
  handled via TOKEN_AUTOFREE_KW and TOKEN_ALLOW_LEAKS_KW.

--------------------------------------------------------------------------------
MEMORY DIRECTIVES
--------------------------------------------------------------------------------

  #autoFree true;    (DEFAULT)
      - Compiler runs escape analysis on every alloc'd pointer, list, and map.
      - Provably-local pointers get OP_FREE emitted automatically.
      - Runtime (Layer 2) catches anything the compiler missed via OP_EXIT_SCOPE.
      - Leak detection still active at exit.
      - autoFree report printed or written to autoFree.txt on exit.

  #autoFree false;
      - Full manual memory. You free everything.
      - Compiler and runtime auto-free both disabled.
      - Leak detection still runs and reports at exit.
      - Runtime null/double-free/use-after-free safety checks still active.

  #allowMemoryLeaks true;
      REQUIRES #autoFree false declared first.
      Combining with #autoFree true is a COMPILE ERROR.
      - Compile-time detected leaks become warnings instead of errors.
      - Program compiles and runs with leaked pointers.
      - Exit leak report still fires.
      - Use for intentional arena/region allocation patterns.

  #allowMemoryLeaks false;    (DEFAULT)
      Compile-time detected leaks abort compilation.

  Valid combinations:
    (none)                         -> mode both, autoFree true, leaks abort
    #mode static;                  -> strict types, autoFree true, leaks abort
    #mode dynamic;                 -> no types, autoFree true, leaks abort
    #autoFree false;               -> manual memory, leaks abort
    #autoFree false;
    #allowMemoryLeaks true;        -> manual, leaks warn only (arena pattern)
    #autoFree true;
    #allowMemoryLeaks true;        -> COMPILE ERROR

--------------------------------------------------------------------------------
SWITCH BREAK MODE DIRECTIVE
--------------------------------------------------------------------------------

  #switchBreakMode false;    (DEFAULT)
      Auto-break mode. Each matched case runs its body and jumps past the
      switch automatically. No explicit break needed. Like Python/Rust match.

  #switchBreakMode true;
      Fallthrough mode. Like C/Java switch. Once a case matches, execution
      falls through into subsequent case bodies unless break is used.
      Default case always executes last regardless of where it is written.


================================================================================
3. TYPES
================================================================================

  Primitive types (value semantics, copied by value):
    int       64-bit signed integer
    float     64-bit IEEE 754 double precision
    bool      true or false
    null      absence of value / uninitialized pointer

  Reference types (heap-allocated, reference semantics):
    string    UTF-8 string; immutable by default
    list      dynamic array of any values, mixed types OK
    map       hash map; any-type keys and values
    pointer   heap-allocated value, managed via alloc / free

  Internal types (runtime only, not user-declarable):
    range     produced by range() calls, lazy and iterable
    function  user-defined function object
    native    built-in C++ function

  Type names for declarations and function signatures:
    int, float, string, bool, list, map, void
    int*, float*, string*, bool*, list*, map*    (pointer types)

  Default values for typed declarations without initializers:
    int       -> 0        (compiler emits INT_VAL(0))
    float     -> 0.0      (compiler emits FLOAT_VAL(0.0))
    string    -> ""       (compiler emits empty ObjString)
    bool      -> false    (compiler emits OP_FALSE)
    list      -> []       (compiler emits OP_LIST_NEW 0)
    map       -> {}       (compiler emits OP_MAP_NEW 0)
    int*      -> null     (compiler emits OP_NULL)

  CORRECTION from old ref: The old doc said int defaults to null. The
  compiler source (NODE_VAR_DECL default init block) shows int defaults to 0.
  Source code is authoritative.


================================================================================
4. VARIABLES
================================================================================

--------------------------------------------------------------------------------
DECLARATION SYNTAX
--------------------------------------------------------------------------------

  Typed (with or without initializer):
    int x = 42;
    float pi = 3.14;
    string name = "Tantrums";
    bool alive = true;
    list nums = [1, 2, 3];
    map data = {"key": "value"};
    int y;                         -> defaults to 0
    list items;                    -> defaults to []
    map kv;                        -> defaults to {}

  Pointer:
    int* p = alloc int(42);
    int* q;                        -> defaults to null

  Dynamic (untyped) -- only valid in #mode both or #mode dynamic:
    x = 42;
    name = "hello";

  In #mode both, bare x = 5; creates an IMPLICIT GLOBAL variable.
  In #mode static, bare assignment to any undeclared name is a hard error.

--------------------------------------------------------------------------------
SCOPING
--------------------------------------------------------------------------------

  Lexical scope. Blocks { } create new scopes.
  Duplicate declaration in the same scope is a compile error.
  Shadowing a built-in name generates a compiler warning.
  Unused local variables generate a compiler warning on scope exit.

--------------------------------------------------------------------------------
TYPE COERCION ON ASSIGNMENT
--------------------------------------------------------------------------------

  When a typed variable is declared, the value is auto-coerced (OP_CAST):
    int x = 3.9;          -> x = 3  (truncation, not rounding)
    float f = 5;          -> f = 5.0
    string s = 42;        -> s = "42"
    bool b = 0;           -> b = false
    bool b2 = "hello";    -> b2 = true  (non-empty, non-whitespace string)

  Coercion does NOT apply to pointer types or alloc init values.


================================================================================
5. FUNCTIONS
================================================================================

  Declared with the tantrum keyword.
  All functions must be defined at top level (global scope only).
  Executable statements at top level outside functions are a COMPILE ERROR.
  Every program MUST define a main() function. It is called automatically
  after all global declarations are processed.

--------------------------------------------------------------------------------
DECLARATION SYNTAX
--------------------------------------------------------------------------------

  Fully typed:
    tantrum int add(int a, int b)
    {
        return (a + b);
    }

  Void:
    tantrum void greet(string name)
    {
        print("Hello " + name);
    }

  Pointer return:
    tantrum int* makeValue()
    {
        int* p = alloc int(42);
        return (p);
    }

  Dynamic (no type annotations):
    tantrum multiply(a, b)
    {
        return (a * b);
    }

  Mixed (typed params, no return type):
    tantrum greetUser(string name)
    {
        print("Hi " + name);
    }

  Entry point:
    tantrum void main()
    {
        print("Hello, World!");
    }

--------------------------------------------------------------------------------
RETURN SYNTAX
--------------------------------------------------------------------------------

  Return values MUST be wrapped in parentheses. Enforced by the parser.
    return (value);    <- correct
    return value;      <- PARSE ERROR: "Return value must be enclosed in ()"

  Bare return (no value):
    return;            <- valid, pushes null

  Implicit return: falling off the end emits OP_NULL + OP_RETURN internally.
  In #mode static this is a hard error for non-void typed functions.
  In #mode both it is a warning.

--------------------------------------------------------------------------------
STATIC MODE FUNCTION RULES
--------------------------------------------------------------------------------

  1. Every function must declare a return type or void (main() exempt).
  2. void functions must not return a value.
  3. Non-void functions must have at least one return statement.
  4. All code paths in non-void functions must return a value.
  5. Return type must match declared type (hard error).
  6. void function calls cannot be used in expressions.
  7. All parameters must have type annotations (main() exempt).
  8. Duplicate function names are a compile error in ALL modes.

--------------------------------------------------------------------------------
FORWARD REFERENCES & PRE-SCANNING
--------------------------------------------------------------------------------

  The compiler pre-scans ALL function signatures before compiling anything.
  Forward references are fully supported: call a function defined later
  in the file or in an imported file without issues.

  Arity errors detected at compile time in #mode both and #mode static.
  Parameter type mismatches checked at compile time when types are known.
  Calls to undefined functions are compile errors in all modes.
  Max 16 typed parameters tracked per function for compile-time checking.
  Max 256 function signatures total (MAX_FUNC_SIGS in compiler.cpp).

--------------------------------------------------------------------------------
DISCARDED POINTER RETURN WARNING
--------------------------------------------------------------------------------

  If a pointer-returning function is called and the return value is not
  assigned to anything, the compiler emits:
    [Tantrums Warning] line N: pointer return value discarded -- potential leak.


================================================================================
6. OPERATORS & EXPRESSIONS
================================================================================

  Arithmetic:
    +    addition; also string concat (auto-converts other side to string)
    -    subtraction; unary negation
    *    multiplication; pointer dereference (unary *p)
    /    int/int -> int; float/anything -> float
    %    modulo (int operands only; whole-number floats coerced to int)

  Comparison:
    ==   equal
    !=   not equal
    <    less than
    >    greater than
    <=   less than or equal
    >=   greater than or equal
    =>   alias for >= (normalized by parser)
    =<   alias for <= (normalized by parser)

  Logical:
    &&   AND (short-circuit: right not evaluated if left is false)
    and  alias for &&
    ||   OR  (short-circuit: right not evaluated if left is true)
    or   alias for ||
    !    NOT (operand must be bool; runtime error otherwise)

  Compound assignment (all desugared to binary + assign by parser):
    +=  -=  *=  /=  %=

  Increment / Decrement:
    i++   postfix: returns OLD value, then increments
    i--   postfix: returns OLD value, then decrements
    ++i   prefix:  increments first, expression is NEW value
    --i   prefix:  decrements first, expression is NEW value
    Prefix forms desugared by parser to: i = i +/- 1

  Pointer:
    *p    dereference (read or write through pointer)
    &     TOKEN_AMPERSAND is lexed but NOT implemented as address-of.
          Do NOT use & until formally supported.

  String auto-concat:
    "age: " + 25          -> "age: 25"
    "pi = " + 3.14        -> "pi = 3.14"
    "alive: " + true      -> "alive: true"
    "val: " + null        -> "val: null"
    Trailing zeros stripped from floats: 3.10 -> "3.1"

  List/range concatenation:
    [1, 2] + [3, 4]       -> [1, 2, 3, 4]
    range(3) + [10, 11]   -> [0, 1, 2, 10, 11]
    Either side can be list or range; result is always a new list.

  Division / modulo by zero:
    Literal divisor 0 at compile time -> compile error.
    Runtime zero divisor               -> catchable runtime error.

  Operator precedence (highest to lowest):
    1. Unary: ! - * (negate/deref)
    2. Postfix: i++ i--
    3. * / %
    4. + -
    5. < > <= >= => =<
    6. == !=
    7. && / and
    8. || / or
    9. = += -= *= /= %=   (right-associative)


================================================================================
7. CONTROL FLOW
================================================================================

--------------------------------------------------------------------------------
IF / ELSE IF / ELSE
--------------------------------------------------------------------------------

  if (condition)
  {
      // then branch
  }
  else if (other_condition)
  {
      // else-if branch
  }
  else
  {
      // else branch
  }

  Conditions MUST evaluate to bool at runtime. A non-bool in JUMP_IF_FALSE:
    [Tantrums Runtime Error] Condition must be a boolean, got <type>.

  Single-statement bodies without braces are supported.

--------------------------------------------------------------------------------
WHILE LOOP
--------------------------------------------------------------------------------

  while (condition)
  {
      // body
  }

  break;      exits the loop
  continue;   jumps back to condition check

--------------------------------------------------------------------------------
FOR-IN LOOP
--------------------------------------------------------------------------------

  for i in range(N)
  for i in range(start, end)
  for i in range(start, end, step)
  for ch in "Hello"
  for item in [1, 2, 3]
  for key in myMap

  range() returns a range object (lazy). Elements computed by OP_FOR_IN_STEP.
  No intermediate list allocated. Iterating a string yields single characters.
  Iterating a map yields keys in internal hash table order (NOT insertion order).
  The loop variable is local to the loop body.
  Hidden vars $iter, $len, $idx are internal and invisible to user code.

  break;      exits the loop
  continue;   skips to next iteration

  Unreachable code after return/throw inside a block triggers a warning.

--------------------------------------------------------------------------------
BREAK & CONTINUE
--------------------------------------------------------------------------------

  Work inside while loops, for-in loops, and switch statements.
  Using outside a loop/switch is a compile error.
  Both correctly unwind nested scopes before jumping.
  Max 64 break or 64 continue jumps per loop construct (hard compiler limit).


================================================================================
8. SWITCH STATEMENT
================================================================================

  switch (expression)
  {
      case (value1) { // body }
      case (value2) { // body }
      default        { // body }
  }

  - Subject expression evaluated once, stored in hidden local $sw.
  - case values compared with == (value equality, same as OP_EQ).
  - default is the fallback: runs only if no case matched.
    Always treated as last regardless of where it appears in the source.
  - Multiple default cases is a compile error.
  - break is supported inside case bodies in both switch modes.

  Auto-break mode (default, #switchBreakMode false):
    Matched case runs then jumps past the switch. No break needed.

  Fallthrough mode (#switchBreakMode true):
    Matched case and all subsequent cases run unless break is used.

  Example (auto-break):
    switch (x)
    {
        case (1) { print("one"); }
        case (2) { print("two"); }
        default  { print("other"); }
    }

  Example (fallthrough):
    #switchBreakMode true;
    switch (x)
    {
        case (1) { print("one"); }    // falls into two without break
        case (2) { print("two"); break; }
        default  { print("other"); }
    }

9. MANUAL MEMORY MANAGEMENT
================================================================================

  alloc type(value)     allocate a value on the heap, return a typed pointer
  free ptr;             release the heap allocation

  Creating and using:
    int* p = alloc int(42);    // allocate int = 42
    print(*p);                 // dereference read -> 42
    *p = 99;                   // dereference write
    free p;                    // release

  Aliasing:
    int* q = p;                // q and p point to same allocation
    // p is now considered escaped -- auto-free will NOT apply

  Passing:
    foo(p);                    // p marked escaped
    append(list, p);           // p marked escaped
    myMap["key"] = p;          // p marked escaped

  Returning:
    tantrum int* makeNum(int val)
    {
        int* p = alloc int(val);
        return (p);            // p escapes, auto-free skipped
    }

  NOT supported:
    &x          address-of (not implemented)
    p + 1       pointer arithmetic
    (int*)p     type casting between pointer types

--------------------------------------------------------------------------------
RUNTIME SAFETY CHECKS (always active, cannot be disabled)
--------------------------------------------------------------------------------

  Use-after-free / null dereference:
    [Tantrums Runtime Error] Null pointer dereference on <type>* pointer!

  Double-free:
    [Tantrums Runtime Error] Double-free detected: pointer has already been freed.

  All pointer errors include a full stack trace (function + line number).
  All pointer errors are catchable inside try blocks.

--------------------------------------------------------------------------------
POINTER SIZE IN REPORTS
--------------------------------------------------------------------------------

  Leak and auto-free reports show alloc size as sizeof(ObjPointer) + sizeof(Value).
  On 64-bit systems this is typically 96 bytes per allocation.

  manage pointers, lists, and maps that are provably local. Two-layer system:

--------------------------------------------------------------------------------
LAYER 1 -- COMPILE-TIME (AST walk)
--------------------------------------------------------------------------------

  After a local alloc is declared, the compiler walks remaining statements

    2. Passed as a function argument

--------------------------------------------------------------------------------
THROW
--------------------------------------------------------------------------------

  throw "error message";     // throw a string value
  throw expression;          // throw any value

  Runtime errors (null deref, division by zero, etc.) are catchable when
  inside a try block. The error message string becomes the catch variable.

  use helper.42AHH;                // legacy bare form (same dir only)

  Imports must appear at the top of the file, before any code.

  How it works:
  - Imported file is lexed, parsed, top-level declarations injected inline.
  - The imported file's own #mode, #autoFree, #allowMemoryLeaks apply
    independently to its declarations (each node tagged with source directives).
  - Duplicate imports of the same file are silently ignored.
  - Circular imports are prevented.
  - File not found -> compile error:
      [Line N] Import Error: Cannot find 'path/to/file.42AHH'.

  Import resolution:
  - Quoted paths resolved relative to the importing file's directory.
  - Bare paths also resolved relative to the importing file's directory.
  - Absolute paths used as-is (Unix / Windows both supported).

  On success:
    [Tantrums] Imported 'helper.42AHH' (N declarations, mode: both)

  Imported functions are pre-scanned before compilation for type checking.
  Max 64 imports per file (hard-coded array in compile_source).

14. BUILT-IN FUNCTIONS
================================================================================

  print(value, ...)
      Print one or more values separated by spaces, followed by newline.
      Any type accepted. Lists: [a, b, c]. Maps: <map>. Ranges: [0, 1, 2, ...].
      Pointers: <ptr>. Floats have trailing zeros stripped: 3.10 -> "3.1".

  input("prompt")
      Print prompt (no newline), read a line from stdin.
      Return type depends on the receiving variable's declared type:
        x = input("?");             -> string (default)
        string s = input("?");      -> string
        int n = input("?");         -> parsed via strtoll
        float f = input("?");       -> parsed via strtod
        bool b = input("?");        -> "true"  -> true
                                       "false" -> false
                                       whitespace-only or empty -> false
                                       any other non-empty string -> true
      Coercion happens via OP_CAST after the raw input() call.

  len(value)
      Returns length as int:
        len("hello")       -> 5
        len([1, 2, 3])     -> 3
        len({"a": 1})      -> 1
        len(range(5))      -> 5
        len(other)         -> 0

  range(end)                    -> lazy range [0, 1, ..., end-1]
  range(start, end)             -> lazy range [start, ..., end-1]
  range(start, end, step)       -> step can be any non-zero integer
      Returns a range object. Step 0 is guarded against in native_range.
      Negative steps work: range(10, 0, -1) -> 10, 9, ..., 1
      Ranges are iterable, indexable, and concatenatable with + and lists.

  type(value)
      Returns type name as string:
        "int", "float", "bool", "null", "string", "list", "map",
        "function", "native", "pointer", "range"

  append(list, value)
      Append value to list in-place. Returns null.
      Appending a pointer to a list marks that pointer as escaped.

17. TYPE SYSTEM & MODES
================================================================================

--------------------------------------------------------------------------------
TYPE INFERENCE (compile-time, best-effort)
--------------------------------------------------------------------------------

  Returns nullptr (unknown/dynamic) when type cannot be determined.
  When nullptr, type checks are skipped (dynamic pass-through).

  Inferred types:
    42               -> "int"
    3.14             -> "float"
    "hello"          -> "string"
    true / false     -> "bool"
    null             -> "null"
    [1,2,3]          -> "list"
    {"k":"v"}        -> "map"
    Variable x       -> from locals table; nullptr if global/unknown
    foo()            -> return type from signature table; nullptr if unknown
    a + b (strings)  -> "string"
    a + b (float)    -> "float"
    a + b (int+int)  -> "int"
    Comparison ops   -> "bool"
    !x               -> "bool"
    *p (int*)        -> "int"

--------------------------------------------------------------------------------
TYPE COMPATIBILITY RULES
--------------------------------------------------------------------------------

  Compatible:
    Either side is nullptr (dynamic)  -> always OK
    Types are identical
    float expected, int actual        -> OK (int promotes to float)
    pointer expected, null actual     -> OK (null is valid pointer)

  Incompatible:
    int* and float*                   -> pointer types must match exactly
    pointer and non-pointer           -> never compatible
    string and int                    -> no implicit conversion

--------------------------------------------------------------------------------
RETURN TYPE ENFORCEMENT BY MODE
--------------------------------------------------------------------------------

  #mode static:
    Mismatch              -> compile error
    Missing return path   -> compile error
    void returning value  -> compile error

  #mode both:
    Mismatch (inferable)  -> WARNING only
    Missing return path   -> WARNING only
    void returning value  -> still a compile error

  #mode dynamic:
    Explicit annotation present but return missing -> WARNING
    Otherwise no enforcement

18. COMPILE-TIME DIAGNOSTICS
================================================================================

  ERRORS (abort compilation):
    Type errors (#mode static and applicable #mode both cases)
    Duplicate function name (all modes)
    Duplicate variable in same scope
    Division by zero (literal operand)
    Memory leak detected (#autoFree false, #allowMemoryLeaks false)
    alloc result assigned to non-pointer typed variable
    alloc type mismatch (e.g. int* p = alloc float(...))
    Call to undefined function
    Arity mismatch on function call
    return / throw outside function (#mode static)
    break / continue outside loop/switch
    Directive ordering violations
    #allowMemoryLeaks true combined with #autoFree true
    Executable code at top level outside functions
    Duplicate directive declarations

  WARNINGS (print to stderr, do not abort):
    Unused local variable
    Shadowed built-in name
    Unreachable code after return/throw
    Pointer return value discarded
    Return type mismatch (#mode both, when inferable)
    Non-void function missing return on all paths (#mode both)
    Memory leak with #allowMemoryLeaks true

  NOTES (stdout, suppressible with --no-autofree-notes):
    [Tantrums] note: auto-freed 'x' at line N (provably local)

20. NATIVE EXECUTABLE FORMAT
================================================================================

  Tantrums now compiles entirely to native executables via an LLVM backend.
  Legacy bytecode (.42ass files) is no longer used or generated by the primary
  pipeline.

  Native executables are standard platform binaries (e.g., ELF on Linux, PE on
  Windows).
  Memory usage is generally a fraction of interpretation overhead.

  Loading a legacy .42ass file is not supported in the standard native builder.
=======
20. BYTECODE FORMAT (.42ass)
================================================================================

  Header:
    4 bytes  magic identifier (BYTECODE_MAGIC from bytecode_file.h)
    1 byte   version field (BYTECODE_VERSION)
    1 byte   global_allow_leaks flag (0 or 1)
    1 byte   global_autofree flag (0 or 1)

  Body: recursive function encoding, starting from the top-level script.

  Per function:
    u32    name length (0 = anonymous/script)
    bytes  name chars
    u32    arity
    u32    constant pool count
    [constants]
      u8   tag: 0=int, 1=float, 2=string, 3=bool(true), 4=bool(false),
                5=null, 6=function (recursive)
      data (tag-dependent lengths)
    u32    bytecode length
    bytes  raw bytecode
    u32    line info count (= bytecode length)
    i32[]  line number per byte (little-endian signed 32-bit)

  All multi-byte integers: little-endian. Floats: IEEE 754 double (8 bytes).
  Loading a file with the wrong version -> error and refusal to run.
